# n8n Forms Chat UX

Use this skill when humans are directly interacting with the workflow.

## Quick workflow

1. Define the user interaction contract.
2. Minimize ambiguity in input collection.
3. Design explicit success, failure, and waiting states.
4. Handle validation and escalation gracefully.
5. Protect user trust by making workflow limitations visible.

## Core rules

- UX failures in automations become operational failures.
- User-facing intake should validate early and explain clearly.
- Chat and form experiences need a fallback path to a human or a clearer next step.
- AI answers should cite workflow confidence, not bluff.
