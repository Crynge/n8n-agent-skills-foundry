# n8n Enterprise Admin

Use this skill when the problem is organizational, not just workflow-local.

## Quick workflow

1. Identify project, role, and environment boundaries.
2. Map who can edit, run, share, and deploy.
3. Decide secret source and governance path.
4. Review source control and environment promotion.
5. Document operational ownership and break-glass procedures.

## Core rules

- Workflow design without ownership design creates hidden operational debt.
- Project boundaries and secret boundaries should align.
- Enterprise features are part of runtime behavior, not just admin convenience.
- Governance should reduce risk without blocking iteration.
